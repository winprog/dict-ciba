# Dict-CiBa - 智能屏幕取词翻译工具

一个基于Python开发的智能屏幕取词翻译工具，支持鼠标悬停取词、自动翻译和智能窗口管理。

## 项目描述

Dict-CiBa是一个轻量级的桌面翻译工具，专门为需要频繁查阅英文单词含义的用户设计。通过智能的屏幕取词技术，用户可以快速获取鼠标悬停位置的单词翻译，无需手动输入或切换窗口。

## 主要功能

### 🎯 智能屏幕取词 - **重大改进**
- **安全取词机制**：使用X11 PRIMARY选择，彻底避免干扰剪贴板
- **用户可控取词**：默认禁用鼠标悬停取词，用户按需启用
- **智能标点清理**：自动清理单词前后的标点符号
- **多源取词支持**：支持选中文本和剪贴板内容
- **零干扰设计**：不会影响命令行程序和其他应用程序的正常使用

### 🌐 金山词霸集成
- **权威词典**：集成金山词霸API，提供准确的单词释义
- **多维度信息**：包含音标、发音、词性、例句等完整信息
- **TTS音频播放**：支持单词发音和例句朗读播放

### 🧠 智能状态管理 - **新增**
- **独立状态机**：专门的单词状态管理器类，分离关注点
- **智能单词比较**：清理标点符号后比较，准确识别相同单词
- **状态跟踪**：完整记录单词切换历史和捕获时间
- **调试支持**：详细的调试信息和状态查询

### 🪟 智能窗口管理
- **智能定位**：窗口自动显示在远离取词位置的对角线区域
- **自动隐藏**：窗口失去焦点后自动隐藏，减少视觉干扰
- **临时置顶**：取词时窗口临时置顶，方便查看结果
- **单词切换检测**：相同单词不重复显示窗口

### 🔊 TTS音频播放 - **新增**
- **多格式支持**：使用ffplay统一支持MP3、WAV等多种音频格式
- **网络流播放**：直接播放HTTP音频URL，无需下载文件
- **一键播放**：单词发音、例句朗读都有直观的播放按钮
- **智能格式识别**：自动识别金山词霸TTS音频格式
- **双重播放机制**：主备播放方式确保成功率

### ⚡ 高效用户体验
- **无干扰操作**：取词过程不影响当前工作流程
- **快速响应**：毫秒级取词和翻译响应
- **自定义配置**：支持窗口透明度、置顶状态等个性化设置
- **架构优化**：模块化设计，便于维护和扩展


3. **单词状态管理器 (word_state_manager.py)** - **新增**
   - 独立的单词状态管理类

## 使用方法

### 基本操作流程

1. **启动应用**

   ```bash
   python3 dictionary_gui.py
   ```
2. **启用取词模式**

   - 点击"启动取词模式"按钮
   - 状态指示灯变为绿色表示取词服务已启动
   - **默认设置**：鼠标悬停取词功能默认禁用，避免干扰其他程序
3. **开始取词**

   - **推荐方式**：用鼠标选中需要翻译的文本（安全，不干扰剪贴板）
   - **可选方式**：如需鼠标悬停取词，请勾选"启用鼠标悬停取词"选项
   - 系统自动捕获单词并显示翻译结果
4. **窗口控制**

   - **置顶显示**: 临时将窗口置于最上层
   - **隐藏窗口**: 最小化窗口到任务栏
   - **自动隐藏**: 窗口失去焦点后自动隐藏

### 安全取词机制说明

#### 🛡️ 分层取词策略（重大改进）

**第一层：PRIMARY选择（推荐）**
- ✅ **零干扰**：使用X11 PRIMARY选择机制，不干扰CLIPBOARD剪贴板
- ✅ **实时获取**：直接获取鼠标选中的文本内容
- ✅ **并发安全**：用户可同时使用Ctrl+C复制和鼠标选中查询

**第二层：双击选择（备用）**
- ⚠️ **温和操作**：模拟鼠标双击选中单词
- ⚠️ **再次尝试**：双击后重新获取PRIMARY选择

**第三层：Ctrl+C（最后手段）**
- ❌ **干扰剪贴板**：仅在上述方法失败时使用
- ❌ **恢复机制**：使用后会尝试恢复用户剪贴板内容

#### 🎯 用户可控设置

**默认安全设置**：
- 鼠标悬停自动取词：**禁用**
- 仅支持用户主动选中文本取词
- 确保不会干扰命令行程序和其他应用程序

**按需启用**：
- 如需鼠标悬停功能，可手动勾选"启用鼠标悬停取词"
- 建议在非敏感环境中使用

### 高级功能

#### 智能窗口定位

- 窗口会根据取词位置自动调整显示位置
- 左上角取词 → 窗口显示在右下角
- 右下角取词 → 窗口显示在左上角

#### 单词切换检测

- 相同单词不会重复显示窗口
- 只有新单词才会触发完整的取词流程

#### 标点符号清理

- 自动清理 `"hello"`, `'world'`, `（example）` 等格式的标点
- 确保查询的单词格式正确

#### TTS音频播放

- **单词发音播放**：点击单词旁边的EN/AM按钮播放英式/美式发音
- **TTS播放**：点击🔊 TTS按钮播放文本转语音
- **例句朗读**：点击例句旁的🔊按钮播放例句发音
- **格式兼容**：自动支持MP3和WAV格式的音频文件
- **静默播放**：后台播放，不影响当前工作流程

**播放按钮说明**：
- 🎵 **EN按钮**：英式发音 (British English)
- 🎵 **AM按钮**：美式发音 (American English)  
- 🔊 **TTS按钮**：文本转语音 (Text-to-Speech)
- 🔊 **例句按钮**：例句朗读播放

## 重大改进：安全取词机制

### 🚀 技术突破

本次更新彻底解决了Dict-CiBa可能干扰其他应用程序的问题，特别是命令行程序。通过引入**X11 PRIMARY选择机制**，实现了真正的零干扰取词体验。

### 🔧 技术实现细节

#### X11选择机制原理

在X Window系统中，有三种主要的选择（selection）：

1. **PRIMARY选择**：鼠标选中的文本（自动更新，不干扰剪贴板）
2. **CLIPBOARD选择**：Ctrl+C复制的文本（用户剪贴板）
3. **SECONDARY选择**：较少使用

**改进前的问题**：
- 使用Ctrl+C模拟操作，会覆盖用户剪贴板
- 干扰命令行程序和其他应用程序
- 用户无法同时使用Ctrl+C和取词功能

**改进后的优势**：
- ✅ **独立操作**：PRIMARY和CLIPBOARD选择完全独立
- ✅ **零干扰**：取词过程不影响用户剪贴板操作
- ✅ **并发安全**：用户可以同时使用Ctrl+C和鼠标选中查询

#### 分层取词策略实现

```python
def get_word_at_position(x, y):
    """获取鼠标位置的单词（安全版，使用PRIMARY选择）"""
    # 第一层：PRIMARY选择（推荐，零干扰）
    selected_text = get_selected_text_primary()
    if selected_text:
        return selected_text
    
    # 第二层：双击选择（备用，温和操作）
    # 第三层：Ctrl+C（最后手段，有干扰但会恢复）
```
